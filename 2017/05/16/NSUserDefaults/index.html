NSUserDefaults

简介

根据官方文档，翻译一下：类 NSUserDefaults 为与系统之间的交互提供了一个编程接口。系统允许应用程序去定制自己的行为来满足用户的喜好。比如说，你可以允许用户去决定文档自动保存的频率。应用程序通过在 用户默认的数据库中设置value 来记录这些喜好。这些参数在启用应用程序的时候，将被作为默认值。这些偏好设置记录在应用包的plist文件中。

举个例子：app有一个弹红包的功能，只能在用户首次进入app的时候弹出。如何判断 用户是首次进入app呢？我们可以设置一个字段，当app打开时，取出该字段的值，判断是否是首次进入，判断完毕后，将该字段置为false，并存入到 NSUserDefaults。

特点： 适合 存储 轻量级的本地数据

使用

NSUserDefaults 是iOS系统提供的一个单例类，获取方式如下：

NSUserDefaults 单例是以key-value形式存储一系列的偏好设置。
存取数据分别通过setObject:forKey和objectForKey:来存储/获取。这些对象将存储到相应的plist文件中。

plist 存储类型有以下几种：

  • NSData
  • NSString
  • NSNumber
  • NSDate
  • NSArray
  • NSDictionary

如果是想存储 plist 文件不支持的类型，eg：图片，则可以先将该对象归档为 NSData 类型，然后再存入到 plist 文件中。

对于 NSArray 和 NSDictionary ，他们的存储类型也应该是上述范围包括的。

NSUserDefaults 返回的value是 不可变 的，即使你设置的value是个可变类型。

NSUserDefaults是线程安全的。

-synchronize is deprecated and will be marked with the NS_DEPRECATED macro in a future release.

-synchronize blocks the calling thread until all in-progress set operations have completed. This is no longer necessary. Replacements for previous uses of -synchronize depend on what the intent of calling synchronize was. If you synchronized…

  • …before reading in order to fetch updated values: remove the synchronize call
  • …after writing in order to notify another program to read: the other program can use KVO to observe the default without needing to notify
  • …before exiting in a non-app (command line tool, agent, or daemon) process: call CFPreferencesAppSynchronize(kCFPreferencesCurrentApplication)
  • …for any other reason: remove

我们在 set值后，并不会立即同步，可以调用 synchronize 去立即同步磁盘上的文件。并不建议这么做，这是一个性能问题。因为 iOS 自己会在合适的时期去 同步。

域

NSUserDefaults 数据库中其实是由多个层级的 域 组成，默认包含 5个Domain 。域 不仅仅用于简单的信息存储，还由其他的设计用途。

  • NSArgumentDomain:参数域，优先级最高，所有的命令行参数都会存放在这个域中
  • Application:应用域
  • NSGlobalDomain:全局域
  • Languages:语言域，地区、日期等
  • NSRegistrationDomain:注册域，优先级最低

app在使用 setObject:forKey: 的时候，都是在 Application 中存储。

默认情况下， Application 是空的，没有key也没有value。当应用第一次设置某项用户偏好设置的value时，相应的 value 会通过 key 加入 Application 。当通过 NSUserDefaults 获取某项用户偏好设置的 value 时， NSUserDefaults 会先在 Application 中查找，如果找到了 value ， NSUserDefaults 就会返回这个 value 。如果没有找到， NSUserDefaults 就会在 NSRegistrationDomain 中查找并返回默认值。

使用 registerDefaults

在获取 value 之前，先调用 registerDefaults 设置默认值，但是， registerDefaults 设置的默认值是 不会持久化存储 的，也就意味着每次启动app时，都要设置一遍。

注意：需要在每次启动 app 并且没有在 user defaults 中读取数据的时候调用以上方法，因为 registerDefaults: 不能把这些默认数据存储到硬盘上，所以 application:didFinishLaunchingWithOptions 是最合适的地方。

在 application:didFinishLaunchingWithOptions 地方去设置默认值，在其他地方使用时，取出值，并 setObject，这时 key 和 value 已经 存储到 Application 域中，下次可 正常使用。

Languages

每次在修改设备的语言设置时，系统都会在 Languages 中设置相应的字段，这个字段的 key 为 AppleLanguages 。

我们也可以通过 NSArgumentDomain 命令行的方式，直接修改结果，-AppleLanguages (zh-Hans) （因为命令行域的优先级最高）
