基础多线程

发表于 2017-09-19 | 分类于 iOS笔记 |

介绍

进程

  1. 进程之间 互相 不可访问，有 各自的 上下文
  2. 进程 5种状态：新建、就绪、运行、阻塞、终止

eg： xcode

线程

  1. 进程的基本执行单元
  2. 进程的所有任务 都在 线程中 执行

eg: xcode中的 代码编辑、文件的保存、编译、下载任务

多线程

适用场景：

  1. 网络请求
  2. 图片加载（先读缓存，如果没有，则发起网络请求，写入缓存）
  3. 文件处理
  4. 数据存储
  5. 任务执行

执行方式： 串行、并行

实现原理：

  1. 单核os中，时间片、轮转调用算法，宏观并行，微观串行
  2. 多核os，真正的并行

优点：

  1. 简化 编程模型
  2. 轻量级
  3. 提高执行效率
  4. 提高资源利用率

缺点：

  1. 增加程序复杂性（资源共享）
  2. 占用内存空间
  3. 增大 cpu调度开销（cpu来回切换）

多线程实现技术方案

4套线程方案：pThread、NSThread、GCD、NSOperation

GCD

苹果为了多核而提出，自动管理线程生命周期。使用c语言，但引入了block，开发方便。

任务：同步、异步，区别在于，会不会 阻塞当前线程。

队列：串行、并行。队列中优先级越高，越有可能先执行

NSOperation

GCD的一种封装，抽象的基类，需要使用 子类。

两种方式：

  1. 系统已经提供的，NSInvocationOptionation & NSBlockOperation
  2. 自定义类，继承 NSOperation

NSOperationQueue：队列，可以理解为 线程池。
addOperation
setMaxConcurrentOperationCount

状态：
ready、cancelled（如果是正在执行的任务，则不会被取消，仍会执行）、executing、finished、asynchronous（判断任务是并发，还是非并发）

依赖：addDependency，任务之间的相互依赖

NSInvocationOptionation

查看控制台，发现 在 主线程 中执行，是因为 使用的是 start 方法，该方法会 阻塞当前 线程。可以将其 放在 dispatch_async 中

NSBlockOperation

同样，也是 阻塞当前线程。

如果采用异步方式呢？ 引入 NSOperationQueue。

NSOperationQueue

一般是将 NSOperationQueue 和 NSInvocationOptionation、NSBlockOperation 一起使用。

主要修改点：将 operation 的启动方式进行修改，采用 [queue addOperation:] 的方式。

自定义Operation

继承自 NSOperation, 重写 main 或者 start 方法即可。

如果设置了 queue 的 setMaxConcurrentOperationCount的值，则 异步并发的线程数 = 该值，执行完 某个 任务，才会开启 下一项 任务。
