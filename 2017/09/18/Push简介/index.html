Push简介

发表于 2017-09-18 | 分类于 iOS笔记 |

Push工作原理

塞班系统 push 有 3 个服务器，而 苹果push 只需要 1 个服务器，且 进行了 各种优化，相比之下，更省电省流量。

iOS 的 push通道 是一个 独立的 长链接 通道，这个通道 是由 iOS系统 独立维护。

每个手机系统 在 开机之后，会有 一个后台的长链接，连到苹果的服务器。如下图 所示：

手机系统 启动 或 打开 app，会生成 唯一的一个 token。

这个 token 可以发送到 自己的服务器，自己的服务器 获取到这个 token 后，并不能直接 推送到 手机，因为 push通道 是由 苹果 维护的。

所以，我们需要将 token+push内容，推到 苹果的通道 APNs，APNs 再根据 token 定位到 设备，将 push内容 推给 设备，设备 根据 token 找到 app，将 通知 push到 具体的app。如图所示：

Local Push

与 remote Push 相对应（APNs），不需要 搭建 服务器。单一的网络通道。

基础用法

例如，有一个 记录 用户打开app 记录日期 的功能，如果用户 1天没有打开app，可以采用local Push去提醒用户。

提高日活。

Local + Remote Push

配合 Remote Push，主要原因 是 网络 问题。一般 先 local Push，如果收到 remote，则用 remote Push。

Local Push 通道更快。

iOS7 之前，苹果在app进入后台时，允许开启一个 后台任务，此时app可以继续运行 10 分钟，iOS7之后，时间缩短到 3分钟。

如何 启用 后台模式 ？

在上述 启动 后台任务 后，我们可以将 服务端的push，用 local push 进行 推送。

Push业务功能

payload：

  1. 携带业务数据
  2. json格式
  3. 有长度限制，iOS8之前，限制在256Byte，

eg：一个日记 push，点击 push信息后，进入到 写日记vc，需要自定义协议

寻找 push推送 的入口，有 2 个，

- (BOOL)application:(UIApplication *)application didFinishLaunchingWithOptions:(NSDictionary *)launchOptions

- (void)application:(UIApplication *)application didReceiveRemoteNotification:(NSDictionary *)userInfo，该方法表明，app处于后台 3分钟

didFinishLaunchingWithOptions 在这个方法里面，有时候需要注意一下，因为推送会立即调用，一些ui操作可能还没有处理，可以延迟几秒钟。

iOS7下的静默push

允许服务器向客户端发起一个静默push，唤起app，保持 30s，app收到push时，没有任何操作，也没有下拉框，仅仅是为了唤起app。

与 后台任务的区别：

后台任务 是 app进入后台时，开启的 保持 3分钟

静默push 是 将 app 唤醒，保持30秒，并不能总是被唤醒，被kill的就不可以

eg：通过 静默push 唤醒 app，执行后台任务，下载内容，通过local push，发送通知 给用户

- (void)application:(UIApplication *)application didReceiveRemoteNotification:(NSDictionary *)userInfo fetchCompletionHandler:(void (^)(UIBackgroundFetchResult))completionHandler

iOS8 Push交互

变更点：

  1. 开启方式：iOS8需要先用setting去换取权限
  2. 交互：支持交互功能，内容展示+下拉动作

VOIP Push

语音和视频电话，eg：QQ或微信中的语音和视频。

iOS8之前，如果想要保持程序在后台一直活跃，需要申请权限，程序常驻后台，这样才能收到电话，后台保持长链接，这样子会费电费流量。

iOS8引入pushKit框架，一种新的push通知类型，即 VOIP Push。

程序 不需要 一直在后台常驻，当有 voip push 时，唤醒程序，30s，在这30s内 可以从 服务器 获取 新数据，然后 再 生成 local push 去通知用户。

与普通push的区别：默认行为里面时不会弹出通知的。

相当于 与 普通push同级别，需要专门申请证书。

只能被 VOIP类的应用 使用，审核也非常严格。

remote push 不能被修改，local push可以修改。
